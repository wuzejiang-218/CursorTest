============================================================
  浏览器提示「无法访问 / 拒绝连接」时 —— 按下面做
============================================================

原因：本机 8000 端口上没有在跑后端程序（服务没启动或已关掉）。

------------------------------------------------------------
正确步骤（缺一不可）
------------------------------------------------------------

1. 双击运行：  启动后端.bat
   （或进入 backend 文件夹，双击 start.bat）

2. 会弹出一个黑色命令行窗口。
   必须等到出现这一行（或类似字样）：
      Uvicorn running on http://127.0.0.1:8000

3. 再打开浏览器，地址里的端口必须和命令行里一致！
   若启动的是 --port 8000，则用：
      http://127.0.0.1:8000/
   若启动的是 --port 8001，则用：
      http://127.0.0.1:8001/
   （用 8000 去连 8001 的服务会连不上。）

   测试页面若用「本地文件」打开，已自动尝试 8000、8001、8002；
   也可: test-platform.html?port=8001 指定端口。

4. 使用期间不要关闭那个黑色窗口；关掉窗口 = 停止服务 = 浏览器又会拒绝连接。

------------------------------------------------------------
若黑窗口一闪就消失
------------------------------------------------------------

说明启动报错。请这样操作：

1. 按 Win+R，输入 cmd 回车
2. 依次输入（每行回车）：

   cd /d d:\cursor\backend
   python -m pip install -r requirements.txt
   python -m uvicorn main:app --host 127.0.0.1 --port 8000

3. 把最后几行红色/英文报错复制下来排查（或发给他人帮看）。

------------------------------------------------------------
若提示端口被占用
------------------------------------------------------------

关掉占用 8000 的程序，或改用别的端口，例如 8001：

   python -m uvicorn main:app --host 127.0.0.1 --port 8001

浏览器则访问：http://127.0.0.1:8001/
