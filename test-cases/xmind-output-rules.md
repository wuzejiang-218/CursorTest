# 测试案例 XMind 输出规范

> **适用范围：** `test-cases/` 下所有 `build_xmind.py` 生成的 `.xmind` 脑图。  
> **可执行明细：** 仍以同目录 `*-测试用例.md`、`cases/*.md` 为准（含 `TC-*` 编号）。

## 原则

| 文档 | 用途 | 是否使用 TC 编号 |
|------|------|------------------|
| `*.md` 测试用例表 | 执行、记录、追溯 | **是** |
| `*.xmind` 脑图 | 评审、宣讲、结构梳理 | **否** |

## XMind 节点命名

**禁止**在脑图节点标题中出现：

- `TC-UI-001`、`TC-NEG-001`、`TC-FCN-KO-X-FACT-001` 等用例编号
- `FACT` / `TIME` / `NEG` 等仅内部缩写（无业务含义时）

**推荐**结构：

```
根主题（版本 + 功能）
├── 1. 模块名（如「产品详情 UI」）
│   ├── 场景：Limit Price 展示
│   │   ├── 前置：…
│   │   ├── 步骤：…
│   │   └── 预期：…
│   └── 场景：…
└── 2. 下一模块
```

**推荐用语：**

- 模块层：业务域（Web 分享、权限、触发事件名）
- 场景层：**动词 + 对象**（仅 UF 不同时拦截、点击图标查看摘要）
- 叶子层：`前置：` / `步骤：` / `预期：` 前缀（可选）

## 与 Markdown 的对应

- 脑图场景名应能与 MD 中某条用例 **语义对应**，但 **不写编号**。
- 需在 MD 中追溯时，在模块 README 注明「脑图见 xmind，执行见 md」。

## 生成方式

各模块目录执行：

```bash
python build_xmind.py
```

公共逻辑见 [`_xmind_common.py`](./_xmind_common.py)。
