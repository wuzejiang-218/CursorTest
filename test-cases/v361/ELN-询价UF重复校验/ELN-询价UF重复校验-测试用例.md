# ELN 询价前端优化 · 除 UF 外条款相同拦截 — 测试用例（可执行）

| 项目 | 内容 |
|------|------|
| **需求摘要** | **ELN** 多行询价提交时：若多行 **除 UF (%) 外其余条款完全相同**，前端 **拦截 Submit** 并提示 |
| **提示文案（英）** | `Quotations with identical terms except for the UF are not supported.` |
| **提示文案（中）** | `不支持除了UF之外其他条款都相同的询价` |
| **产品** | EasyConnect · Quotation · **Add New Quotation** · **ELN** Tab |
| **参考原型** | [需求原型-ELN-UF校验.png](./需求原型-ELN-UF校验.png) |
| **版本** | v3.61 |
| **编写日期** | 2026-05-19 |

---

## 1. 校验规则（测试判定标准）

| 条件 | 是否拦截 |
|------|----------|
| ≥2 行 ELN，参与比对的字段 **全部相同**，且 **仅 UF (%) 不同** | **是** |
| 仅 1 行 | **否** |
| 除 UF 外还有任意字段不同（如 Strike、Gross Margin、Underlying 等） | **否** |
| 多行 **完全一致**（含 UF 相同） | **否**（走原有重复/业务规则，本需求不额外拦截） |
| 非 ELN 品类 Tab | **不适用**本规则 |

**参与比对字段（与页面表格列一致，以产品字段表为准）：**  
Currency、Underlying、Solve For、Strike (%)、KO Type、Tenor (m)、Gross Margin (%)、Issue Date、Final Valuation Date、Maturity Date 等；**UF (%) 不参与「相同」判定，仅作为允许不同的唯一字段。**

---

## 2. 测试范围

| 在范围 | 不在范围 |
|--------|----------|
| ELN Tab 新增多行询价 Submit 前校验 | FCN / Sharkfin 等其他 Tab |
| 中英文错误提示 | 后端入库后校验（本需求为 **前端**） |
| 修改行数据后再次 Submit | 导入模板批量询价（若无 ELN 多行则 N/A） |

## 3. 测试数据准备

| 数据编号 | 行数 | 设计要点 | 预期 |
|----------|------|----------|------|
| D-BLOCK-01 | 2 | 除 UF 外全同；UF 行1=1，行2=1.2 | 拦截 |
| D-BLOCK-02 | 3 | 三行仅 UF 分别为 1 / 1.2 / 1.5，其余同 | 拦截 |
| D-PASS-01 | 2 | 除 UF 外全同；**UF 也相同** | 不因此规则拦截 |
| D-PASS-02 | 2 | UF 不同，且 **Strike** 不同 | 不拦截 |
| D-PASS-03 | 2 | UF 不同，且 **Gross Margin** 不同 | 不拦截（非「仅 UF 不同」） |
| D-PASS-04 | 2 | UF 不同，且 **Underlying** 不同 | 不拦截 |
| D-PASS-05 | 1 | 单行 ELN | 不拦截 |
| D-OTHER-01 | 2 | FCN Tab 两行仅 UF 不同 | 不触发 ELN 规则 |

**基准行示例（对齐原型）：**  
Currency=USD，Underlying=3888.HK，Solve For=Note Price (%)，Strike=95，KO Type=NA，Tenor=6，Gross Margin=1，日期三列相同。

## 6. 通过准则

- P0 **100%** 通过。
- 「仅 UF 不同」仍 **Submit 成功** → **Blocker**。
- 其它字段不同却被误拦 → **Major**。
- 中英文文案与需求 **不一致** → **Major**。
