## 5.1 同步回复申请状态

| 状态 | 含义 |
|------|------|
| **待确认** | 已创建，等待 Issuer 确认 |
| **已确认** | Issuer 已确认，价格与最优价已更新 |
| **已拒绝** | Issuer 拒绝 |
| **已失效** | 超过机构/ Issuer 有效期、询价关闭、配置不匹配、旧回复被覆盖等 |

## 5.2 发起条件（手动 + 自动共用）

除 [02-机构配置.md](./02-机构配置.md)、[03-Issuer端配置.md](./03-Issuer端配置.md) 中模式差异外，创建申请须同时满足：

1. 询价处于可比价/可跟进状态；
2. 目标 Issuer = 机构配置的 **Matched Last Look Issuer**；
3. Issuer 侧存在该询价机构的 **Matched Last Look Buyside** 行且已启用；
4. 存在当前有效价格回复，且价格 ≠ 本询价当前最优价；
5. 无同「询价 + Issuer + 价格回复」的待确认/已确认申请；
6. **手动模式**：当前用户 ∈ **User authorized to initiate last look**；
7. 若配置 **Best Price Difference Value**，价差未超阈值（仅自动发起）；
8. Issuer **Price Validity Period**（若配置）未过期。

## 5.3 一次性约束

- **粒度**：`询价 ID` + `Issuer ID` + `价格回复 ID`；
- **规则**：同一组合仅允许一次有效申请（已确认后不可对同一回复再发起）；
- **新回复**：Issuer 待确认期间提交新回复 → 原申请**已失效**，可按新回复重新评估自动/手动发起。

## 5.4 确认成功后更新

1. 更新被申请价格回复为确认价（见 [06-约束与边界.md](./06-约束与边界.md) §6.3）；
2. **本次询价最优价格** = 该 Issuer 确认后回复价；
3. 记录 `trigger_mode`（机构手动/机构自动）、`confirm_channel`（Issuer 邮件/自动/手工）、机构与 Issuer 配置快照 ID。

## 5.5 拒绝与失效

| 事件 | 处理 |
|------|------|
| Issuer 拒绝 | 已拒绝；价格与最优价不变 |
| 超过机构 **Last Look Validity Period** | 已失效 |
| 超过 Issuer **Price Validity Period** | 已失效或不可确认 |
| 询价关闭 | 所有待确认 → 已失效 |
| 机构 Issuer 与 Issuer Buyside 未配对 | 不创建申请或已失效 |
| 价差 > Best Price Difference Value | **不自动发起**（见 [02-机构配置.md](./02-机构配置.md) §2.3） |
| 确认时最优价已变化 | 跟**确认时刻最新最优价**（见 §6.3） |
