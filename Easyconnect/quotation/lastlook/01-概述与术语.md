# 01 概述与术语

[← 返回知识库首页](./README.md)

---

## 1.1 背景

机构用户在 EasyConnect 中向多个 Issuer 发起债券/产品询价。各 Issuer 独立回复价格后，系统在同一笔询价维度内计算并展示**当前最优价格**及其对应 Issuer。

当某 Issuer 的回复价格**不是**当前最优价时，机构侧可通过 **Lastlook** 向该 Issuer 发起**同步回复申请**，请求其将报价调整为与当前最优价一致。申请可由机构用户**手动发起**，也可在机构配置为**自动模式**时由系统按条件自动发起（**机构 Organization → Last Look Setting**）。Issuer 通过 **Issuer → Last Look Setting** 中配置的 **Reply Last Look Mode**（邮件 / 自动 / 手工）确认后，该 Issuer 的回复价更新为最优价，且**本次询价的最优价格**变更为该 Issuer 确认后的价格。机构与 Issuer 配置须 **Matched Issuer ↔ Matched Buyside** 双向配对方可生效。

## 1.2 术语定义

| 术语 | 定义 |
|------|------|
| **询价（Inquiry）** | 机构用户创建的一笔询价单，包含标的、期限、方向、参与 Issuer 列表等基础信息。 |
| **价格回复（Price Reply）** | 某 Issuer 针对该笔询价提交的一条完整报价记录（含价格、量级、有效期等字段，以产品字段定义为准）。 |
| **Issuer** | 收到询价并可在系统中/邮件等渠道回复价格的报价方。 |
| **机构（Organization）** | 使用 EasyConnect 的机构主体；Last Look 相关配置在机构维度维护（如 CAI HK）。 |
| **机构用户** | 隶属于机构的系统用户；在手动模式下可发起 Lastlook；受「授权发起用户」配置约束。 |
| **最优价格（Best Price）** | 在同一笔询价下，按系统统一比价规则从**所有 Issuer 当前有效价格回复**中计算得出的最优报价。 |
| **当前最优 Issuer** | 持有当前最优价格的那条价格回复所属 Issuer。 |
| **Lastlook** | 询价改价/比价跟进功能，展示各 Issuer 回复与最优价对比，并承载同步回复申请的创建与状态跟踪。 |
| **同步回复申请（Sync Reply Request）** | 针对某一 Issuer **整笔价格回复**发起的、请求将报价同步为当前最优价的一次性申请。 |
| **触发 Last Look 模式（Triggering Last Look Mode）** | 机构配置项：**Manual（手动）** 或 **Automatic（自动）**，决定申请由人触发还是系统触发。 |
| **Matched Last Look Issuer** | 机构配置中绑定的、适用本套 Last Look 规则的 Issuer（可与询价参与 Issuer 匹配校验）。 |
| **Matched Last Look Buyside** | Issuer 配置中绑定的、与本 Issuer 开展 Last Look 的机构（Buyside）。 |
| **Reply Last Look Mode** | Issuer 侧回复模式：邮件 / 自动 / 手工，决定如何确认同步回复申请。 |
| **Match Price Time** | Issuer 侧与指定 Buyside 的匹配价格时间窗口。 |
