## 6.1 最优价格比较范围

- **范围**：同一笔询价（Inquiry ID）下，所有参与 Issuer 的**当前有效价格回复**。
- **排除**：已作废、已过期、草稿未提交回复（以询价模块状态为准）。
- **比价规则**：与询价引擎一致（买/卖方向决定更优价高低）；Lastlook 不单独定义公式。

## 6.2 并发与时效

- 同一「询价 + Issuer + 价格回复」最多 **1 条**待确认申请；
- 申请有效期 = 机构 **Last Look Validity Period**（如 20 分钟），且受 Issuer **Match Price Time** / **Price Validity Period** 约束（取较严者，待产品确认）；
- 自动与手动同时满足时，后端幂等，仅创建一条申请。

## 6.3 价格一致性（确认时刻）

**创建申请时记录：**

- `target_best_price`：创建时最优价快照；
- `target_best_issuer_id`：创建时最优 Issuer（审计）；
- `applicant_reply_id`：被申请的价格回复 ID。

**Issuer 确认时：**

1. 读取确认时刻 `latest_best_price`；
2. **默认策略**：以 `latest_best_price` 为同步目标（跟最新最优价），审计记录与快照差异；
3. 确认成功后：Issuer 回复价 = 同步目标价；**本询价最优价** = 该 Issuer 确认后回复价。

## 6.4 权限

| 角色 | 权限 |
|------|------|
| 机构管理员 | 维护机构 Last Look Setting（[02-机构配置.md](./02-机构配置.md)） |
| Issuer 管理员 | 维护 Issuer Last Look Setting（[03-Issuer端配置.md](./03-Issuer端配置.md)） |
| 机构用户（授权） | Manual 模式下发起；查看 Lastlook |
| 机构用户（未授权） | Manual 模式下不可发起 |
| 系统 | Automatic 模式下按机构配置自动发起；校验双向配对 |
| Issuer | 按 Reply Last Look Mode 确认/拒绝申请 |

## 6.5 非目标（Out of Scope）

- 不改变询价创建与首次回复流程；
- 机构用户不能代替 Issuer 直接改价；
- 不支持对「已确认」申请撤回后，对同一价格回复再次发起。
