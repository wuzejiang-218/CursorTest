# PoC 范围冻结与验收指标

**版本**: 1.0  
**状态**: 已冻结（PoC 用）  
**周期**: 3 个月

## 1. 业务目标

- 将高频回归测试人力投入降低 **≥30%**（以人工基线对比）。
- 将核心链路缺陷发现前移到提测前（缺陷发现阶段前移占比可统计）。

## 2. 交付范围（已冻结：3 条核心金融流程）

| ID | 流程名称 | 说明 | PoC 边界 |
|----|----------|------|----------|
| F1 | 授信申请 | 从进件到授信结果（含必要字段校验） | 仅测试环境；不含第三方征信实时联调（可用 Mock） |
| F2 | 风控决策 | 规则引擎/评分卡输出与决策码一致性 | 规则版本固定为 PoC 基线版本号 |
| F3 | 放款核验 | 放款前校验（额度、状态、合同要素等） | 不落真实放款；以接口与状态机为准 |

超出上表的需求变更须走「范围变更」记录（见文末）。

## 3. 验收指标（量化）

| 指标 | 目标 | 度量方式 |
|------|------|----------|
| 自动生成并执行用例占比 | ≥ 核心回归集的 **40%** | 核心回归集清单行数 vs 智能体生成并执行行数 |
| 核心流程回归耗时 | 较基线下降 **≥50%** | 同环境、同数据规模下流水线端到端耗时 |
| 关键缺陷漏检率 | **不高于**现有人工基线 | 与基线周期内同类缺陷对比 |
| 可追溯性 | **100%** 流水线运行可回放 | 每次运行具备：输入摘要、步骤日志、执行产物、结论与版本号 |

## 4. 非目标（PoC 不做）

- 生产环境全量替代人工测试。
- 全产品线 UI 视觉回归。
- 模型训练与自有基座大模型部署。
